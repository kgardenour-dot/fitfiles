# Fitfiles App Charter

## Purpose

Fitfiles exists to give people a simple, structured place to save and organize workouts they discover across the internet.

It is not a workout tracker.
It is not a coaching platform.
It is not a social fitness feed.

**Fitfiles is a personal workout library.**

Just as recipe apps allow users to save and organize recipes from blogs and social media, Fitfiles allows users to collect and structure workout content from YouTube, Instagram, blogs, PDFs, apps, and anywhere else they train.

**The problem it solves:**

People discover great workouts everywhere, but lose them in bookmarks, screenshots, DMs, notes apps, or endless scrolling.

Fitfiles centralizes those routines into one organized, searchable system.

## Core Principles

### Single Primary Action
Everything revolves around one action: **Save Workout**.

### Organization Over Tracking
Fitfiles stores and structures workouts. It does not measure performance or track progress metrics.

### User-Owned Library
Users build a curated personal collection of routines they actually want to repeat.

### Low Friction
Saving a workout should take seconds. Retrieval should be effortless.

### Calm Utility
No feed. No algorithms. No social noise. Just your workouts.

## Target User

Individuals who:

- Discover workouts across multiple platforms
- Prefer following online routines rather than fixed programs
- Want to reduce scrolling and decision fatigue
- Train consistently but informally
- Value organization and simplicity

## Core Features (MVP)

### 1. Save Workout via Link

Users paste a URL from:

- YouTube
- Instagram
- Blog posts
- Online programs
- PDFs
- Any web source

The app:

- Extracts available metadata (title, thumbnail, domain)
- Allows manual editing
- Saves the workout to the user's library

### 2. Tagging and Categorization

Users can assign structured attributes such as:

- Duration
- Equipment required
- Body focus
- Difficulty
- Custom tags

Tags enable filtering and smart organization without rigid folder constraints.

### 3. Personal Library

The main screen displays saved workouts as organized cards.

Users can:

- Search by title or keyword
- Filter by tags
- Sort by recently added, duration, or favorites
- Quickly open the original source

The library is the heart of the app.

### 4. Collections

Users can create custom collections, similar to recipe collections.

Examples:

- Hotel Room
- Under 20 Minutes
- Upper Body
- No Equipment
- Travel Week

Collections are optional overlays, not rigid folders. A workout can exist in multiple collections.

### 5. Workout Detail View

Each saved item includes:

- Title
- Source link
- Thumbnail (if available)
- Assigned tags
- Personal notes
- "Open Workout" button

Optional lightweight feature:

- "Mark Done" for personal reference only (no analytics).

## What Fitfiles Is Not

- Not a fitness tracker
- Not a calorie counter
- Not a coaching app
- Not a transformation program
- Not a social platform
- Not algorithm-driven

It does not prescribe workouts.
It preserves them.

## Long-Term Vision

Fitfiles may later support:

- Smart collections
- Import from browser bookmarks
- Shared collections (optional)
- Light usage history

But it remains centered on **organization, not performance tracking**.
